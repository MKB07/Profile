# Computational Health Profile

## Motivation

My programming work started from practical biomedical needs. I wanted to create tools that make genetic information easier to structure, visualize and reason about.

I am self-taught in computer science and learned by building, reading documentation, testing ideas, debugging errors, and iterating on concrete tools. AI-assisted coding tools helped me accelerate this process by explaining documentation, suggesting implementation patterns, and supporting debugging. I treat these tools as assistants, not as replacements for biological understanding or critical review.

## Technical Skills

- R and Shiny.
- Python.
- HTML and CSS.
- C.
- Database and spreadsheet workflows: Excel, REDCap, Access.
- Clinical information system support: GLIMS.
- AI-assisted prototyping, code explanation and debugging.

## Application Areas

My computational interests include:

- genetic pedigree software;
- biomedical information interfaces;
- phenotype annotation;
- family-history representation;
- clinical genetics education tools;
- decision-support prototypes that keep human review central.

## Positioning

My profile is not a conventional software-engineering-only profile. It is a domain-expert profile strengthened by self-directed software development and AI-assisted learning. This is useful in projects where biological validity, usability and scientific caution matter as much as implementation.
